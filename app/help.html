Getting Started

Go into preferences and add the folders that hold your media collections. Goon will scan the folders and generate thumbnails for your collection. NOTE: If you have a very large collection you may want to choose one sub folder at a time. See tips below.

Note: Scanning is ssssllllooowww currently as Goon has to load every image and video to make thumbnails. Fortunately you should only have to do this once after which Goon will just use thumbnails it previously created. Also you can quit and Goon will scan the rest next time you run it.

Password

You may enter a password in the preferences. If one exists you'll be prompted for it when the program starts.

Hidden UI

When viewing an image much of the UI is hidden until you hover the mouse over it. There are next and previous areas on the left and right. If you're viewing a video there is a video control near the bottom. There are exit, zoom, and rotate buttons on the top right. And there is the filename of the current item at the top.

App Rotation

There are 2 types of rotation in Goon. One is the common "rotate an image" type of rotation. If you are viewing an image and press the rotate icon or press a rotate key the image will rotate.

The other type is app rotation which rotates the entire UI.

App Rotation is designed for laptops for viewing images in portrait mode. To use it, first make sure a pane with thumbnails has the focus then press a rotate key. The UI will rotate as above. Then hold your laptop as below.

Zoom/Stretch Modes

Pressing zoom cycles though the following modes

constrain
scale down so the image fits the view but never larger than its actual size
stretch
scale up or down to fit the view
cover
scales so that the entire view is covered by the image even of part of that image ends up offscreen
actual size
Just what it says.
fit width
scale until the image fills the width of the view
fit height
scale until the image fills the height of the view

Filters

When a thumbnail pane has the focus you can type filters in the toolbar. Just typing letters matches the entire path. So for example gif matchs both spinning-n.gif and gifs/nerf.jpg. .gif would match both christmas.gifts/nuts.png and neptune.gif

Words that end in a colon : are considered a special filter.

width:
height:
aspect:
size:

all of these take a comparision and a number. Examples

width:>1000 height:<=512 aspect:> size:<200k

aspect can also be aspect:landscape and aspect:portrait which are just synomyms for aspect:>1 and aspect:<1 respectively.

size and can take the following case insensitive suffixes

k, kb m, mb g, gb t, tb

examples

size:>100mb size:>1.5gb
date:

takes a date in YYYY-MM-DD format as in

date:<2015 date:>2018-02-15
folder:
dir:
dirname:

work the same as above but only match the folder portion so folder:nuts would match nuts/almond.jpg but not collections/nuts.gif

basename:
filename:

Do the same for just the base name of the file so filename:new would match animals/newts.mp4 but not newspapers/post.png.

bad:

Will show files that could not be opened for some reason.

type:

Filters on mime-types so for example type:video would show all videos and type:image/gif would show all gifs.

Comparision Operators

> greater than
>= greater than or equal to
< less than
<= less than or equal to
== equal to
=== equal to
!= not equal to
!== not equal to

Comparison operators only work with width:, height:, aspect:, size:, and date:.

Adding Folders in a Big Collection

Goon is alpha software. Scanning can take a long time and use too much memory. One way around this is to add subfolders. For example maybe you have 1000 images in a folder called 'Newts' and another 1000 in a folder called 'Nebulas' and both of those folders in are a parent folder called 'N' something like

N | +--Newts | +--Nebulas

In preferences you could add folder 'N' and all 2000 files would be scanned. Or you could add separately first folder 'Newts' then later at some other time folder `Nebulas` and get effectively the same result but only scanning half of your images at a time.

If later you want to add folder 'N' and remove folders 'Newts' and 'Nebulas' it's important to add 'N' *before* removing the other two folders. The moment you remove a folder all thumbnails below will be removed if they are not covered in some other way. By adding 'N' before removing the subfolders Goon can keep using your already scanned thumbnails.

Note: really large collections can be hard to navigate. Hopefully the planned organizing features will help. Until then see below for other options.

Launching from the Command Line

You can launch Goon from the command line and pass in one or more folders to view. This is one simple way to pair down your collection.

Example:

"path-to-electron" path/to/newts path/to/nails

Would view only files in those 2 folders. When running Electron this way the folders specified in your preferences are ignored.

Bugs and Workarounds

Adding network folders on MacOS

There is apparently a bug in Electron that prevents browsing the contents of network shares. Until that bug is fixed you'll need to add network shares by manually editing your preferences file.

Inside you should see a folder's entry something like this if you have no folders

"folders": [],

Or something like this if multiple folders exist

"folders": [ "some/path/to/a/folder", "some/path/to/another/folder" ],

To add a network share add a folder path on its own line surrounded by double quotes

"folders": [ "path/to/network/share/folder", "some/path/to/a/folder", "some/path/to/another/folder" ],

Save the file and re-start Goon.

Refreshing Network Shares

Watching for changes on local hard drive generally works but watching for change on remote or network storages is often problematic. If Goon scans all your folders each time you run it so if Goon does not notice your changes live try quiting and starting it again.
